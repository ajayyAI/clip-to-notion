# Chrome Web Store Submission Checklist

## Store assets
- [ ] Extension icon set included (`16`, `32`, `48`, `128`)
- [ ] Screenshots captured from real extension usage
- [ ] Optional promo tile / marquee assets prepared

## Listing content
- [ ] Product name finalized
- [ ] Short description finalized (<= 132 chars)
- [ ] Detailed description finalized
- [ ] Category selected (`Productivity`)
- [ ] Support contact email/URL added
- [ ] Privacy policy URL added (must be publicly hosted)

## Privacy and data disclosures
- [ ] Single purpose statement entered
- [ ] Data handling disclosures completed in CWS form
- [ ] Permission justifications match actual behavior
- [ ] No undeclared data collection or analytics

## Technical checks
- [ ] `manifest.json` references valid icon paths
- [ ] Permissions are minimal and justified
- [ ] Extension loads cleanly in `chrome://extensions`
- [ ] Save flow tested on timeline + post-detail pages
- [ ] Notion duplicate detection validated

## Packaging
- [ ] Create upload zip from project root
- [ ] Exclude nonessential files if needed (optional)
- [ ] Upload package and complete review questionnaire
